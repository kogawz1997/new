# Release Notes v1.0.0

วันที่อ้างอิง: **2026-03-15**

release นี้เป็นจุดที่เอกสารและการตรวจระบบเริ่มอ้างกลับไปหาไฟล์, test, และ artifact ได้ชัดขึ้น

## สรุป

- split runtime ชัดขึ้นเป็น `bot / worker / watcher / admin web / player portal / console-agent`
- delivery runtime แยก execution backend เป็น `rcon` กับ `agent` ชัดเจนขึ้น
- admin web มีเครื่องมือสำหรับ preflight, simulator, timeline, capability test, audit, restore, และ security events
- verification ของ repo ถูกผูกกับ CI artifacts มากขึ้น แทนการอ้างตัวเลขแบบ hardcode ในเอกสาร

## รายการเปลี่ยนแปลง

### 1. Delivery runtime ถูกทำให้ debug ได้จริงขึ้น

- order มี `executionMode`, `backend`, `commandPath`, `retryCount`
- มี preflight ก่อน enqueue เมื่อเป็น `agent mode`
- มี timeline / step log / status history / evidence bundle
- มี circuit breaker และ failover policy ฝั่ง agent

เวลาส่งของพัง สามารถตอบได้ชัดขึ้นว่า:
- พังที่ queue, worker, agent, command execution, หรือ verification
- order นี้วิ่งผ่าน `rcon` หรือ `agent`
- retry ไปกี่ครั้งแล้ว

### 2. Admin / auth boundary

- มี Discord SSO, 2FA, step-up auth, session revoke, security event trail
- restore/config change/bulk operations อยู่ในกลุ่ม route ที่ต้องระวังมากขึ้น
- shared-origin deployment ถูกผลักให้แยก `admin` กับ `player` ชัดขึ้น

### 3. Restore safety

- restore preview
- preview token + expiry
- maintenance gate
- rollback backup
- compatibility layer สำหรับ backup shape รุ่นเก่า

### 4. Verification

- source of truth หลักคือ CI workflow และ artifacts
- มี `verification-summary.json` และ `verification-summary.md`
- มี smoke, readiness, security-check, doctor logs เก็บเป็น artifact

## ผลกระทบต่อผู้ดูแลระบบ

สิ่งที่ผู้ดูแลระบบจะเห็นใน release นี้:
- debug order delivery ง่ายขึ้น
- รู้เร็วขึ้นว่า runtime ไหนตายหรือไม่พร้อม
- restore มี guardrail มากขึ้นก่อนเขียนทับ production
- auth flow ฝั่ง admin มีชั้นป้องกันมากขึ้น

สิ่งที่ยังต้องดูแลเอง:
- Windows session ถ้าใช้ `agent mode`
- SCUM admin client และ admin channel state
- OAuth / split-origin / reverse proxy / secret rotation

## Known Limitations In This Release

- `agent mode` ยังเปราะกับ patch เกมและสภาพแวดล้อมของ desktop automation
- `admin web` ยังไม่ครบทุก setting ที่มีใน env/config
- SQLite ยังเหมาะกับ single-host มากกว่าหลายเครื่องหลาย worker
- restore flow ยังต้อง manual confirmation บางจุด
- screenshot dashboard จริงและ demo GIF ยังไม่ได้ถูกเก็บไว้ใน repo

## หมายเหตุการ deploy

release นี้ควร deploy ด้วยเงื่อนไขต่อไปนี้:
- `NODE_ENV=production`
- `PERSIST_REQUIRE_DB=true`
- `PERSIST_LEGACY_SNAPSHOTS=false`
- split-origin สำหรับ admin/player
- รัน `doctor`, `security:check`, `readiness:prod`, `smoke:postdeploy`

ถ้าใช้ `agent mode`:
- เปิด SCUM admin client ค้างไว้
- อย่า lock Windows session
- อย่าเปิด delivery worker ซ้ำทั้ง `bot` และ `worker`
