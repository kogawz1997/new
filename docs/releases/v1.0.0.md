# Release Notes v1.0.0

วันที่อ้างอิง: **2026-03-15**

release นี้เป็นจุดที่ระบบเริ่มมีฐาน runtime, delivery tooling, auth hardening, และ verification flow ที่ชัดขึ้นพอสำหรับใช้งานจริงบน split runtime

## สรุป

- แยก runtime ชัดเป็น `bot / worker / watcher / admin web / player portal / console-agent`
- delivery runtime แยก execution backend เป็น `rcon` กับ `agent`
- admin web มี preflight, simulator, timeline, capability test, audit, restore preview, และ security events
- verification ของ repo ผูกกับ CI artifacts มากขึ้น
- มีเส้นทาง cut over ไป PostgreSQL และ runtime บนเครื่องนี้ใช้ PostgreSQL แล้ว

## การเปลี่ยนหลัก

### Delivery

- order มี `executionMode`, `backend`, `commandPath`, `retryCount`
- มี preflight ก่อน enqueue เมื่อใช้ agent mode
- มี timeline / step log / evidence bundle
- มี agent circuit breaker และ failover policy

### Admin / security

- DB login, Discord SSO, 2FA, step-up auth, session revoke, security event trail
- restore/config change/bulk actions อยู่ใน route ที่เข้มขึ้น

### Persistence

- เพิ่ม helper สำหรับ PostgreSQL local cluster
- เพิ่ม provider-aware Prisma generate/migrate
- เพิ่ม cutover script จาก SQLite ไป PostgreSQL
- test runner แยก schema/database ให้ตาม provider ที่ใช้จริง

### Tenant foundation

- มี tenant-tagged commerce และ delivery rows
- มี tenant config API และ tenant-scoped admin guard
- ยังไม่ใช่ full per-tenant isolation

## ผลกระทบต่อผู้ดูแลระบบ

- debug order delivery ง่ายขึ้น
- แยกได้ว่า order ไหนวิ่งผ่าน `rcon` หรือ `agent`
- restore มี guardrail มากขึ้นก่อนเขียนทับ production
- runtime database ไม่ต้องผูกอยู่กับ SQLite อย่างเดียวอีกแล้ว

## ข้อจำกัดใน release นี้

- agent mode ยังขึ้นกับ Windows session, SCUM admin client, และ patch เกม
- admin web ยังไม่ครอบทุก setting ใน env/config
- tenant isolation ยังไม่ใช่ database-per-tenant
- screenshot dashboard จริงและ demo GIF ยังไม่ถูกเก็บใน repo

## เงื่อนไขการ deploy

- `NODE_ENV=production`
- `PERSIST_REQUIRE_DB=true`
- `PERSIST_LEGACY_SNAPSHOTS=false`
- split-origin สำหรับ admin/player
- รัน `doctor`, `security:check`, `readiness:prod`, `smoke:postdeploy`
