Surface Map

What belongs where

This platform avoids role confusion by keeping responsibilities and risk levels separate.

Owner only

Global health + governance

  • Tenant fleet health and lifecycle
  • Runtime posture for bot, worker, and watcher
  • Security center, audit, and request anomalies
  • Backup, restore, and global config
Tenant only

Scoped server operations

  • Shop and economy control
  • Delivery queue and dead-letter handling
  • Wallet adjustments and scoped config patches
  • Tenant-specific notifications and player support
Player only

Trusted self-service flow

  • Wallet, purchases, and redeem state
  • Profile, Steam linking, and transaction history
  • Mobile-friendly browsing and clear status feedback
  • Legacy community features kept separate from core flows

Runtime posture

This landing page stays documentation-oriented rather than marketing-heavy. It exists to explain the platform shape and current operating model.

1. Owner controls stay separated

Platform-wide configuration and risky recovery flows are kept away from tenant day-to-day operations.

2. Tenant controls stay scoped

Commerce, delivery, and wallet actions remain tied to tenant context instead of a shared global dashboard.

3. Player experience stays focused

Wallet, purchase, redeem, profile, and Steam linking are first-class flows without leaking operational complexity.
